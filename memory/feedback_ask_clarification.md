Before implementing any feature or change, ask clarification questions if anything is ambiguous — behavior, edge cases, UI placement, interactions with existing features, etc.

**Why:** User explicitly requested this to avoid wasted work and misaligned implementations.

**How to apply:** On every implementation request, scan for ambiguities and ask before writing any code. If nothing is ambiguous, proceed directly.
